================================================
GROUP TRIP ORGANIZER - DEPLOYMENT GUIDE
================================================

Your app is ready to deploy! Here's exactly what to do:

STEP 1: Create a GitHub repo
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Go to github.com (sign up if needed)
2. Click "+" → "New repository"
3. Name it: "trip-organizer"
4. Make it Public
5. Click "Create repository"

STEP 2: Push your code to GitHub
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

In your terminal (Mac/Linux) or PowerShell (Windows):

  cd /home/claude/trip-organizer-app
  
  git init
  git add .
  git commit -m "Initial commit: Group Trip Organizer"
  git remote add origin https://github.com/YOUR_USERNAME/trip-organizer.git
  git branch -M main
  git push -u origin main

(Replace YOUR_USERNAME with your actual GitHub username)

STEP 3: Connect Vercel
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Go to vercel.com
2. Sign in with GitHub
3. Click "Add New" → "Project"
4. Select the "trip-organizer" repo
5. Click "Import"
6. Let Vercel auto-detect settings (keep defaults)
7. Click "Deploy"

STEP 4: Add Database
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

After deployment completes:

1. Go to your Vercel project dashboard
2. Click "Storage" tab
3. Click "Create Database" → "Postgres"
4. Name: "trip-organizer-db"
5. Select region (default is fine)
6. Click "Create"

Vercel auto-adds POSTGRES_URL to your env vars. ✓

STEP 5: Done!
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Your live app is at:
  https://trip-organizer.vercel.app

Share this URL with your group!

Everyone who visits can:
✓ Add items in real-time
✓ See everyone's contributions instantly
✓ Check off items as they're acquired
✓ Switch between different trips
✓ No import/export needed — it's all synced!

WHAT'S INCLUDED
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Frontend:
  - Modern UI with categories
  - Real-time sync (updates every 2 sec)
  - Multi-trip support
  - Stats dashboard

Backend:
  - Express.js server
  - REST API for trip data
  - Vercel Postgres database
  - Auto-scales on Vercel

DATABASE SCHEMA
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

trip_items table:
  - id (unique item ID)
  - trip_id (e.g., "moose-25")
  - person (who's bringing it)
  - item (what they're bringing)
  - category (Shared Items, Food, etc.)
  - notes (optional extra details)
  - completed (boolean)
  - created_at (timestamp)

TROUBLESHOOTING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

"Deployment failed"
→ Check that git push succeeded
→ Check you're in the right directory

"Database not connecting"
→ Verify POSTGRES_URL is in Vercel environment
→ Wait 60 seconds after creating database
→ Redeploy from Vercel dashboard

"Items not syncing"
→ Check browser console (F12) for errors
→ Verify database was created
→ Check network tab to see API calls
